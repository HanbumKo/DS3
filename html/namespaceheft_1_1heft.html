def heft.heft.schedule_dag (   dag,

  communication_matrix = C0,

Given an application DAG and a set of matrices specifying PE bandwidth and (task, pe) execution times, computes the HEFT schedule of that DAG onto that set of PEs.

Parameters
daga NetworkX DiGraph instance that contains all the tasks to be scheduled as well as their dependency and communication volume relationships
computation_matrixthe matrix of task to PE computation cost estimates used to inform scheduling decisions
communication_matrixthe matrix of PE to PE communication cost estimates used to inform scheduling decisions
proc_schedulesif existing schedules need to be passed in to HEFT (i.e. we are not scheduling onto an idle system), then they can be passed in as fixed constraints here
time_offsetif the schedule here should be generated relative to some time offset rather than the current instance, specify so here
relabel_nodesif the nodes in the incoming DAG need to be relabeled to not conflict with tasks specified in the proc_schedules constraints, specify so here
rank_metricspecify the RankMetric enum to be used in scheduling this DAG
kwargsoptional kwargs (op_mode - can be used to specify the operating mode, power_dict - contains a matrix of PE to power cost estimates used to inform power-aware scheduling decisions)
