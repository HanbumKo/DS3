7 Steps to include other user-defined DVFS policies:
8 - Define a name for the custom policy, which will be used in the SoC config file and must be applied to all clusters (excluding the memory).
9 - Define an initial frequency for the custom policy in the initialize_frequency method.
10 - Create a method at the end of this file that receives the required parameters and changes the clusters' frequency according to the defined policy.
11 - Add a call for the new DVFS policy in the evaluate_PE method of DTPM.py

52 def ondemand_policy(cluster, PEs, timestamp):
53  '''!
54  Default Linux's ondemand policy.
55  High and low utilization thresholds are configured in config_file.ini (util_high_threshold and util_low_threshold).
56  @param cluster: Cluster object
57  @param PEs: The PEs available in the current SoC
58  @param timestamp: Current timestamp
59  '''
60  # When using ondemand, evaluate the PE utilization and adjust the frequency accordingly
61  utilization = DASH_Sim_utils.get_cluster_utilization(cluster, PEs) * cluster.num_active_cores
62  if utilization <= common.util_high_threshold and utilization >= common.util_low_threshold:
63  # Keep the current frequency
64  DTPM_power_models.keep_frequency(cluster, timestamp)
65  elif utilization > common.util_high_threshold:
66  # Only modify the frequency if the cluster is not being throttled
67  if common.throttling_state == -1:
68  # Set the maximum frequency
69  DTPM_power_models.set_max_frequency(cluster, timestamp)
70  elif utilization < common.util_low_threshold:
71  # Decrease the frequency
72  DTPM_power_models.decrease_frequency(cluster, timestamp)
73  else:
74  print("[E] Error while evaluating the PE utilization in the DVFS module, all test cases must be previously covered")
75  sys.exit()
76  cluster.policy_frequency = cluster.current_frequency
